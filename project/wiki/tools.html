工具系统

发仔的工具系统以 ToolRegistry 为核心，提供统一的工具注册与执行机制。 Agent 在推理过程中可动态选择并调用工具，所有工具调用均经由 ToolRegistry 路由分发。

工具系统概述

工具系统的设计遵循以下原则：

  • 统一注册与执行：所有工具通过 ToolRegistry（registry.py） 集中注册，Agent 只需调用 registry.execute(name, args)，无需感知工具的具体实现。
  • 22+ 内置工具：覆盖文件系统操作、Shell 执行、多数据源查询、飞书协作、 分析辅助、Web 搜索与用户交互等七大类别，开箱即用。
  • 可扩展架构：所有工具均继承自 BaseTool 基类， 新增工具只需实现 execute() 方法并在 tool_setup.py 中注册， 即可立即被 Agent 使用，无需修改核心推理逻辑。

并发执行：当模型在单次响应中返回多个工具调用时， ToolRegistry 使用 asyncio.gather 并发执行所有工具， 显著降低多工具场景下的总体延迟。单个工具失败不会中断其余工具的执行， 错误信息以 tool_result 的形式返回给模型。

工具分类详情

文件系统工具（filesystem.py）

文件系统工具封装了对沙箱工作区内文件与目录的读写操作， Agent 可借助这些工具读取用户上传的数据文件、写入分析结果或浏览目录结构。

工具名 说明
ReadFileTool 读取文件内容
WriteFileTool 写入文件
EditFileTool 编辑指定行
ListDirTool 列出目录内容
GrepTool 搜索文件内容

Shell 工具（shell.py）

Shell 工具提供在隔离沙箱中执行命令的能力，支持 Shell 脚本与 Python 代码的混合执行。 所有执行均在安全受控的容器环境中进行，详见 沙箱环境文档。

工具名 说明
ExecTool 在沙箱中执行 Shell/Python 命令

数据查询工具

数据查询工具对接多个内部与外部数据源，Agent 可通过自然语言推理自动选择合适的数据源并构造查询， 将结构化数据结果注入上下文进行进一步分析。

工具名 数据源 说明
AeolusQueryDataTool Aeolus 字节数据仓库查询
AeolusPreviewDataTool Aeolus 预览数据集
AeolusRunSqlTool Aeolus 执行 SQL
AeolusGetDimValuesTool Aeolus 获取维度值
CisCoreFetchDataTool CIS Core 核心业务数据
LarkSheetFetchDataTool Lark Sheets 飞书表格数据
GetDimensionsTool - 获取维度定义
SentryDataFetchTool Sentry 异常监控数据
BladeOfflineSqlTool Blade 离线 SQL 查询

协作工具（飞书）

协作工具集成飞书（Lark）能力，使 Agent 能够将分析结果以富文本卡片的形式直接发送至飞书频道或会话， 实现分析结论的即时分享与可视化呈现。

工具名 说明
LarkChartTool 创建飞书卡片图表

分析辅助工具

分析辅助工具为 Agent 提供记忆管理、任务调度与技能加载等能力， 帮助 Agent 在长会话中保持上下文连贯、追踪多步任务进度并动态扩展执行能力。

工具名 说明
MemoryTool 持久化 Agent 记忆（memory.json）
SessionSearchTool 搜索历史会话内容
SkillTool 加载并执行动态 Skills
TodoTool 任务待办管理
TaskStoreTool 任务状态存储

Web 工具

Web 工具使 Agent 能够访问互联网，获取最新的财经资讯、市场数据或外部参考资料， 弥补内部数据仓库的时效性不足。

工具名 说明
WebSearchTool 网络搜索
WebFetchTool 抓取网页内容

交互工具

交互工具允许 Agent 在执行过程中主动向用户发起提问， 用于澄清模糊需求、确认关键参数或请求用户授权， 从而在无需中断整体流程的前提下获取必要的补充信息。

工具名 说明
AskUserTool 在执行中向用户提问

如何扩展新工具

得益于 BaseTool 抽象基类与集中注册机制，为发仔添加新工具只需三步， 无需修改 Agent 核心推理逻辑。

步骤一：继承 BaseTool

在 app/agent/tools/ 目录下新建工具文件，创建继承自 BaseTool 的类， 并声明工具名称、描述与参数 Schema： 

from app.agent.tools.base import BaseTool, ToolResult

class MyCustomTool(BaseTool):
    name = "my_custom_tool"
    description = "这里描述工具的功能，供 LLM 理解何时调用它"
    parameters = {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "查询参数说明"
            }
        },
        "required": ["query"]
    }

步骤二：实现 execute() 方法

在工具类中实现异步的 execute() 方法，包含实际的工具逻辑， 并返回 ToolResult 对象： 

    async def execute(self, query: str) -> ToolResult:
        # 在此实现工具逻辑
        result = await some_external_api(query)
        return ToolResult(
            success=True,
            content=str(result)
        )

步骤三：在 tool_setup.py 中注册

打开 app/agent/tool_setup.py，导入新工具类并将其实例注册到 ToolRegistry： 

from app.agent.tools.my_custom_tool import MyCustomTool

def setup_tools(registry: ToolRegistry) -> None:
    # ... 已有工具注册 ...
    registry.register(MyCustomTool())

完成以上三步后，新工具即会自动出现在 LLM 可调用的工具列表中， Agent 将根据用户意图自动决策是否调用该工具。

提示：工具描述（description）和参数说明的质量直接影响 LLM 的工具选择准确率，请尽量用清晰、具体的语言描述工具的用途与参数含义。
