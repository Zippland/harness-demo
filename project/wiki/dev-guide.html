Finance Agent (发仔) · 字节跳动内部项目

开发者入门与贡献指南

本文档面向希望本地运行、调试或向发仔项目贡献代码的开发者，涵盖环境搭建、 快速启动、开发规范和扩展方式等核心内容。建议在阅读本文之前先了解 架构设计 和 Agent 循环， 以便更好地理解各组件的职责边界。

开发环境要求

在开始之前，请确保本地已安装以下依赖：

依赖 版本要求 用途
Python >= 3.11 后端运行时
uv 最新版 Python 包管理器，替代 pip/poetry
Node.js >= 20 LTS 前端构建工具链
pnpm >= 10.29.2 前端包管理器
MongoDB >= 6.0 主数据库，存储会话、工作区等数据
Redis >= 7.0 消息队列与缓存

推荐：使用 Docker Compose 启动 MongoDB 和 Redis。 项目根目录的 docker-compose.dev.yml 提供了开箱即用的本地基础设施配置， 执行 docker compose -f docker-compose.dev.yml up -d 即可完成启动。

快速启动

按照以下步骤克隆仓库并在本地运行完整的开发环境：

1. 克隆仓库

# 克隆主仓库
git clone https://github.com/your-org/finance-agent.git
cd finance-agent

2. 后端：安装依赖

# 使用 uv 安装所有 Python 依赖（包括开发依赖）
uv sync

3. 配置环境变量

# 复制示例配置文件
cp .env.example .env

# 编辑 .env，填入关键配置（MongoDB、Redis、LLM 密钥等）
# 至少需要配置：MONGO_URI、REDIS_URL、LLM_PROVIDER 及对应的 API Key
vim .env

4. 启动后端开发服务

# 仅启动后端 API 服务（热重载）
make serve

# 同时启动飞书 Bot 监听（需配置 LARK_APP_ID / LARK_APP_SECRET）
make serve_with_bot

5. 前端：安装依赖并启动

# 一键安装前端依赖并启动开发服务器（等同于 cd web && pnpm install && pnpm dev）
make fe

启动成功后，前端开发服务器默认监听 http://localhost:5173， 后端 API 默认监听 http://localhost:8000。 浏览器访问前端地址即可开始调试。

关键环境变量

以下是 .env 文件中最常用的配置项。完整字段列表参见仓库根目录的 .env.example 文件。

变量 说明 示例值
LLM_PROVIDER LLM 服务商，决定使用哪个推理后端 openai | volcengine
OPENAI_API_KEY OpenAI API 密钥（LLM_PROVIDER=openai 时必填） sk-...
SANDBOX_PROVIDER 代码沙箱类型，影响 Python 代码执行方式 bytedai | local
MONGO_URI MongoDB 连接字符串 mongodb://localhost:27017/finance_agent
REDIS_URL Redis 连接字符串 redis://localhost:6379/0
LARK_APP_ID 飞书应用 App ID（集成飞书 Bot 时必填） cli_a1b2c3d4e5f6
LARK_APP_SECRET 飞书应用 App Secret（集成飞书 Bot 时必填） xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

本地开发推荐：将 SANDBOX_PROVIDER 设为 local， 无需外部沙箱服务即可在本机直接执行 Python 代码片段，适合快速验证工具逻辑。 生产环境请使用隔离沙箱以保障安全性。

常用开发命令

后端

# 安装 / 同步 Python 依赖
uv sync

# 启动后端开发服务器（支持热重载）
make serve

# 同时启动飞书 Bot
make serve_with_bot

# 代码风格检查（不修改文件）
uv run ruff check .

# 自动格式化代码
uv run ruff format .

# 检查格式是否符合规范（CI 用，不修改文件）
uv run ruff format --check .

前端

# 进入前端目录
cd web

# 安装依赖
pnpm install

# 启动开发服务器（HMR，监听 localhost:5173）
pnpm dev

# 生产构建（输出到 web/dist/）
pnpm build

# ESLint 代码检查
pnpm lint

# 运行单元测试
pnpm test

# TypeScript 类型检查（全量，不生成输出文件）
pnpm typecheck:web

代码规范速查

发仔项目对代码风格有明确约定，提交前请确保代码通过以下规范检查。 详细规范文档参见仓库 docs/CONTRIBUTING.md。

Python 规范

规则 说明 示例
绝对导入 禁止使用相对导入（from . import），一律使用绝对路径导入 from app.core.agent import Agent
类型注解必填 所有函数参数和返回值必须添加类型注解，包括私有方法 def run(self, query: str) -> AgentResult:
类名 PascalCase 类名使用大驼峰命名法 class ToolExecutor:
函数/变量 snake_case 函数名和变量名使用下划线分隔的小写字母 def build_system_prompt():
常量 UPPER_SNAKE_CASE 模块级常量使用全大写下划线命名 MAX_RETRY_COUNT = 3

TypeScript 规范

规则 说明 示例
明确类型定义 使用 type 或 interface 定义所有数据结构 type MessageRole = 'user' | 'assistant'
禁止 any 避免使用 any 类型，使用 unknown 或精确类型替代 const data: unknown = JSON.parse(raw)
类型专用导入 导入仅用于类型标注的符号时，使用 import type import type { Session } from '@/types'

通用规范

  • 函数长度 ≤ 80 行：超过时应拆分为更小的职责单一函数
  • 单一职责：每个函数只做一件事，不混用不相关逻辑
  • 禁止硬编码常量字符串：URL、数据库集合名、错误码等均应提取为具名常量或从配置读取

扩展点说明

发仔采用插件化设计，以下是常见的扩展场景和对应的接入方式：

添加新 Tool

所有工具均继承 BaseTool 抽象类并实现 run() 方法， 完成后在 tool_setup.py 中注册到工具注册表即可。 

# app/tools/my_tool.py
from app.tools.base import BaseTool, ToolResult

class MyTool(BaseTool):
    name = "my_tool"
    description = "工具的功能描述，会注入到 LLM 的 System Prompt"

    async def run(self, param_a: str, param_b: int = 10) -> ToolResult:
        # 实现工具逻辑
        result = do_something(param_a, param_b)
        return ToolResult(output=str(result))

# tool_setup.py — 注册新工具
from app.tools.my_tool import MyTool

def build_tool_registry() -> dict:
    return {
        # ... 现有工具
        MyTool.name: MyTool(),
    }

添加 LLM Provider

继承 LLMProvider 基类并实现 stream() 方法， 在 provider 工厂函数中按 LLM_PROVIDER 环境变量分支注册。 

# app/llm/providers/my_provider.py
from app.llm.base import LLMProvider, StreamChunk
from typing import AsyncIterator

class MyLLMProvider(LLMProvider):
    async def stream(
        self, messages: list[dict], **kwargs
    ) -> AsyncIterator[StreamChunk]:
        # 调用第三方 LLM API，yield StreamChunk
        ...

添加数据源

在 app/infra/ 目录下新建客户端模块，封装数据源的连接与查询逻辑； 再在 app/tools/ 下编写对应的 Tool，通过依赖注入使用该客户端。 保持客户端和 Tool 的职责分离：客户端只负责 I/O，Tool 负责参数校验和结果格式化。

添加 API 路由

在 app/api/routes/ 目录下新建路由文件，定义 FastAPI Router， 然后在 main.py 中通过 app.include_router() 注册。 新增的 API 接口需同步更新 docs/API.md 文档。 

# app/api/routes/my_resource.py
from fastapi import APIRouter

router = APIRouter(prefix="/api/my-resource", tags=["my-resource"])

@router.get("/{resource_id}")
async def get_resource(resource_id: str):
    ...

# main.py
from app.api.routes.my_resource import router as my_resource_router
app.include_router(my_resource_router)

添加 MCP Server

发仔支持通过 Tool Configuration Center（TCC）动态挂载 MCP（Model Context Protocol）Server。 只需在 TCC 的工作区配置中添加对应的 MCP Server 地址和参数，Agent 初始化时会自动发现并加载， 无需修改任何代码。
