API 接口

发仔后端提供完整的 RESTful HTTP 接口与 SSE 流式接口，供 Web 前端、Lark Bot 及外部系统集成调用。 本文档详细说明各接口的路径、参数、认证方式与响应格式。

接口概述

基础前缀

所有接口均以 /api 为路径前缀，例如：

https://<host>/api/sessions
https://<host>/api/sessions/{id}/messages

认证方式

系统使用 JWT Token 进行身份认证，Token 存储于 HttpOnly Cookie 中，浏览器请求时自动携带，无需手动设置请求头。对于非浏览器客户端（如 CLI、Bot）， 也支持通过请求头显式传递： 

Cookie: token=<jwt-token>

# 或（非浏览器客户端）
Authorization: Bearer <jwt-token>

JWT Token 由 Lark OAuth 回调接口（/api/auth/callback）颁发， 有效期默认 7 天，到期后需重新登录。 未携带有效 Token 的请求将返回 401 Unauthorized。

统一响应格式

所有接口（SSE 流式接口除外）均返回如下统一 JSON 结构：

{
  "code": 0,           // 0 = 成功，非 0 = 业务错误
  "message": "ok",     // 成功时为 "ok"，错误时为具体说明
  "data": { ... }      // 业务数据载体，成功时携带返回内容
}

错误码说明

code HTTP 状态码 含义
0 200 成功
401 401 未认证或 Token 已过期
403 403 无权限访问该资源
404 404 资源不存在
409 409 资源冲突（如重复创建）
422 422 请求参数校验失败
500 500 服务内部错误

接口分组

认证接口

发仔使用 Lark（飞书）OAuth 2.0 进行用户身份认证。 用户首次访问时跳转至飞书授权页，授权完成后由回调接口颁发 JWT Token 并写入 HttpOnly Cookie。

方法 路径 说明
GET /api/auth/login 发起 Lark OAuth 跳转，将浏览器重定向至飞书授权页
GET /api/auth/callback OAuth 授权回调，用 code 换取 access_token，签发 JWT 并写入 Cookie
GET /api/profile 获取当前已登录用户的基本信息（需认证）

工作区接口

工作区（Workspace）是用户的顶层数据隔离单元，每个工作区可包含多个会话， 并独立配置数据源权限与 Skill 开关。

方法 路径 说明
POST /api/workspaces 创建新工作区
GET /api/workspaces 获取当前用户所有工作区列表
GET /api/workspaces/{id} 获取指定工作区详情
PUT /api/workspaces/{id} 更新工作区名称或配置
DELETE /api/workspaces/{id} 删除工作区（级联删除其下所有会话与文件）

会话接口

会话（Session）代表一次完整的对话上下文，所有消息均属于特定会话。 会话创建后可持续追加消息，历史记录永久保存于 MongoDB。

方法 路径 说明
POST /api/sessions 在指定工作区下创建新会话
GET /api/sessions 获取当前用户的会话列表（支持按工作区过滤）
GET /api/sessions/{id} 获取指定会话的元数据
DELETE /api/sessions/{id} 删除指定会话及其所有消息与产物文件

消息接口

消息接口是发仔的核心交互入口。发送消息后，系统在后台异步触发 Agent 推理循环， 推理过程中的实时事件通过 SSE 流式接口推送给客户端。

方法 路径 说明
POST /api/sessions/{id}/messages 向指定会话发送用户消息，异步触发 Agent 推理循环
GET /api/sessions/{id}/messages 获取指定会话的历史消息列表（分页）
GET /api/sessions/{id}/stream 建立 SSE 连接，实时接收 Agent 推理事件流

文件接口

文件接口支持用户上传原始数据文件（Excel、CSV、PDF 等）， 以及获取 Agent 在沙箱中生成的产物文件（图表、报告）。 所有文件内容存储于 TOS 对象存储，接口返回签名 URL 以确保安全访问。

方法 路径 说明
POST /api/files 上传文件至 TOS，返回文件 ID 与元数据
POST /api/files/signed_urls 批量获取文件的签名下载 URL（有效期 1 小时）
GET /api/sessions/{id}/files/{path} 从沙箱执行产物目录下载指定路径的文件

Skills 接口

Skills（技能）是对 Agent 能力的模块化扩展，每个 Skill 可独立启用或禁用。 Skills 接口用于查询系统已注册的 Skill 列表，并管理当前工作区的 Skill 开关状态。

方法 路径 说明
GET /api/skills 获取系统所有已注册 Skills 的列表及当前启用状态
PUT /api/skills/{id}/enable 在当前工作区启用指定 Skill
PUT /api/skills/{id}/disable 在当前工作区禁用指定 Skill

数据源接口

数据源接口用于发现和查询系统可用的数据集与数据源类型， 供前端在创建工作区或会话时展示数据选择器。

方法 路径 说明
GET /api/aeolus/datasets 自动检测并返回当前用户有权访问的 Aeolus 数据集列表
GET /api/data-sources 获取系统支持的所有可用数据源类型（如 MySQL、Hive、TOS 等）

分享接口

分享接口允许用户将某次会话的分析结果生成可公开访问的分享链接， 无需登录即可查看分享内容（只读）。

方法 路径 说明
POST /api/shares 为指定会话创建分享 token，返回可公开访问的分享链接
GET /api/shares/{token} 通过分享 token 访问分享会话的消息与产物（无需认证）

SSE 事件类型

当客户端通过 GET /api/sessions/{id}/stream 建立 SSE 连接后， Agent 推理过程中产生的各类事件将以如下格式实时推送： 

data: {"type": "text.delta", "content": "本季度营收", "session_id": "sess_abc123"}

data: {"type": "tool.start", "tool": "python_repl", "args": {"code": "df.describe()"}}

data: {"type": "session.done", "session_id": "sess_abc123"}
事件类型（type） 触发时机 主要字段 说明
session.start Agent 推理循环启动时 session_id 标志本次推理开始，前端可据此清空输入框、显示加载状态
text.delta LLM 流式输出每个文本片段时 content 增量文本内容，前端应将多次 delta 拼接后渲染为完整 Markdown
tool.start Agent 发起工具调用时 tool、args 工具名称与调用参数，前端可据此显示"工具执行中"的状态卡片
tool.done 工具执行完成（成功或失败）后 tool、output、error 工具执行结果摘要；若执行失败，error 字段携带错误信息
session.done Agent 本轮推理全部完成后 session_id 标志本次推理结束，前端应关闭 SSE 连接并恢复输入状态

断点续传：若 SSE 连接意外断开，前端可携带 Last-Event-ID 请求头重新连接，服务端将从断点处继续推送未读事件， 不丢失任何推理过程数据。

请求 / 响应示例

以下为发送消息接口（POST /api/sessions/{id}/messages） 的完整请求与响应示例。

Request

POST /api/sessions/sess_abc123/messages
Content-Type: application/json
Cookie: token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

{
  "content": "请分析本季度各业务线营收趋势，并生成折线图",
  "attachments": [
    {
      "file_id": "file_xyz789",
      "filename": "Q3_revenue.xlsx"
    }
  ]
}

Response

HTTP/1.1 200 OK
Content-Type: application/json

{
  "code": 0,
  "message": "ok",
  "data": {
    "message_id": "msg_def456",
    "session_id": "sess_abc123",
    "role": "user",
    "content": "请分析本季度各业务线营收趋势，并生成折线图",
    "attachments": [
      {
        "file_id": "file_xyz789",
        "filename": "Q3_revenue.xlsx"
      }
    ],
    "created_at": "2024-10-15T09:30:00Z",
    "stream_url": "/api/sessions/sess_abc123/stream"
  }
}

SSE 事件流示例

消息发送成功后，前端通过 data.stream_url 建立 SSE 连接， 将依次收到以下事件： 

data: {"type": "session.start", "session_id": "sess_abc123"}

data: {"type": "tool.start", "tool": "python_repl", "args": {"code": "import pandas as pd\ndf = pd.read_excel('Q3_revenue.xlsx')\ndf.head()"}}

data: {"type": "tool.done", "tool": "python_repl", "output": "业务线A  业务线B  业务线C\n1250万   980万   1430万"}

data: {"type": "text.delta", "content": "根据 Q3 数据分析，"}

data: {"type": "text.delta", "content": "本季度各业务线营收情况如下："}

data: {"type": "tool.start", "tool": "chart_render", "args": {"type": "line", "title": "Q3 各业务线营收趋势"}}

data: {"type": "tool.done", "tool": "chart_render", "output": {"file_id": "file_chart_001", "filename": "revenue_trend.png"}}

data: {"type": "text.delta", "content": "折线图已生成，业务线 C 表现最佳，同比增长 18.5%。"}

data: {"type": "session.done", "session_id": "sess_abc123"}
