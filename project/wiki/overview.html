Finance Agent (发仔) · 字节跳动内部项目

Finance Agent（发仔）是什么？

Finance Agent（发仔）是字节跳动内部研发的财务数据分析服务，基于 AI Agent 架构， 接受自然语言查询，返回财务与运营数据洞察。用户无需掌握 SQL 或编程技能， 即可通过对话方式完成复杂的财务分析任务。

接口形态

发仔提供两种原生接入方式，满足不同使用场景：

  • RESTful API + 前端 Web：标准 HTTP 接口配套 React Web 应用， 支持多轮对话、文件上传下载、流式响应与数据可视化。
  • 飞书 Bot：深度集成字节跳动内部 IM 平台飞书， 用户在飞书群聊或私信中即可直接与发仔交互，无需切换工具。

项目定位：AI Agent 驱动的企业级财务分析平台—— 将大语言模型的自然语言理解能力与结构化财务工具深度融合， 实现从数据查询到报告生成的全流程智能化。

核心能力

🔄

多轮对话 Agent

基于 LangGraph 的流式响应 Agent，支持工具调用与上下文记忆，多轮对话持续推理直至任务完成。

🛡️

沙箱代码执行

集成 ByteAI Sandbox，支持 Python / Shell 安全隔离执行，数据分析代码在受控环境中运行，杜绝安全风险。

🗄️

数据平台集成

原生对接 Aeolus、Coral、Finance Portal、CIS Core、Lark Sheets 等内部数据平台，统一查询入口。

📁

文件管理

支持文件上传与下载，自动解析 Excel、CSV、PDF 等常见财务文件格式，提取结构化数据供 Agent 使用。

⚡

Skills 插件系统

可扩展的工具插件体系，开发者可按规范快速接入新的数据源或分析能力，无需修改核心框架。

💬

飞书 Bot 集成

原生飞书 IM 访问支持，在飞书单聊、群聊中直接发起财务查询，消息卡片展示图表与报告。

🔐

多用户多工作区隔离

每位用户拥有独立工作区与会话空间，数据与文件严格隔离，满足企业级安全与权限管控要求。

项目结构

以下是 finance_agent/ 的主要目录结构及各模块职责说明：

finance_agent/
├── app/                        # 后端主应用（FastAPI）
│   ├── api/                    #   路由层：RESTful 接口定义
│   ├── agent/                  #   Agent 核心：LangGraph 循环、上下文管理
│   ├── tools/                  #   工具注册中心：内置工具与 Skills 插件
│   ├── sandbox/                #   沙箱执行器：ByteAI Sandbox 封装
│   ├── integrations/           #   数据平台集成：Aeolus、Coral、CIS Core 等
│   ├── lark/                   #   飞书 Bot：消息接收、卡片渲染、权限校验
│   ├── models/                 #   数据模型：MongoDB ODM、Pydantic Schema
│   └── core/                   #   基础设施：配置、日志、认证、异常处理
│
├── web/                        # 前端应用（React 19 + Vite）
│   ├── src/
│   │   ├── components/         #   UI 组件库
│   │   ├── pages/              #   页面视图
│   │   ├── stores/             #   Zustand 状态管理
│   │   └── api/                #   前端 API 客户端
│   └── public/                 #   静态资源
│
├── docs/                       # 项目文档（Wiki 源文件）
│
├── cronjob/                    # 定时任务脚本（数据同步、清理任务等）
│
├── scripts/                    # 运维与部署脚本（初始化、迁移、调试工具）
│
└── workspace/                  # 用户工作区（文件上传存储，按用户隔离）
