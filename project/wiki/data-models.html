Finance Agent (发仔) · 字节跳动内部项目

数据模型

本文档详细介绍 Finance Agent（发仔）的数据存储设计，涵盖 MongoDB 集合结构、 Redis 缓存策略以及 TOS 对象存储路径规范。

存储架构概述

发仔采用三层存储架构，每层各司其职，相互协作：

存储层 技术选型 职责
主持久化 MongoDB（ByteDoc） 消息历史、会话元数据、文件元数据、技能注册信息的持久存储
热缓存 / 实时通信 Redis 活跃会话消息热缓存（TTL 2h）、分布式锁、SSE 事件流、任务取消信号
对象存储 TOS（字节跳动对象存储） 用户上传文件、Agent 产物（图表 / 报告）、Skills 技能包

MongoDB：主持久化

MongoDB 使用字节跳动内部的 ByteDoc 托管服务，存储所有结构化业务数据。 核心集合包括：agent_workspaces、agent_sessions、 agent_messages、agent_skills、agent_files、 agent_shares、agent_traces。

Redis：热缓存 + 实时通信

Redis 承担四类职责：

  • 活跃会话消息热缓存：将正在进行中的会话消息列表缓存于 Redis， TTL 设置为 2 小时，避免每次推理都读取 MongoDB，显著降低延迟。
  • 分布式锁：使用 SET NX PX 原语对会话加悲观锁， 保证同一会话同时只有一个写操作，防止消息乱序或重复处理。
  • SSE 事件流：Agent 推理过程中通过 XADD 将 token、 工具调用、工具结果等事件写入 Redis Stream，API 层通过 XREAD BLOCK 消费并推送给前端 EventSource。
  • 取消信号：用户发起取消请求时，向对应 Redis Key 写入取消标志， Agent 循环在每轮推理前检测该信号并提前终止。

TOS：对象存储

TOS 存储所有二进制文件内容，按照固定路径规范组织，确保多租户隔离。 详细路径规范见本文末尾 TOS 存储路径规范。

MongoDB 集合详情

agent_workspaces

工作区是用户与 Agent 协作的独立环境，持久化保存用户文件树与已加载技能的状态。 每个用户可拥有多个工作区，工作区与沙箱实例（sandbox_sid）绑定。

字段说明

字段名 类型 说明
_id ObjectId 工作区唯一标识，MongoDB 自动生成
user_id String 所属用户 ID，来源于 JWT Token
name String 工作区名称，用户自定义，默认为"默认工作区"
sandbox_sid String 绑定的沙箱实例 Session ID，用于代码执行环境恢复
files Map<String, FileEntry> 工作区文件树，key 为相对路径，value 为文件元数据（见下方结构说明）
skills Map<String, SkillEntry> 工作区已配置的技能，key 为 skill_id，value 为启用状态（见下方结构说明）
status String 工作区状态：active（正常）/ archived（已归档）
created_at DateTime 工作区创建时间（UTC）
updated_at DateTime 最后更新时间（UTC），每次文件或技能变更时自动刷新

files 字段结构（path → FileEntry）

files 是一个 Map，key 为文件在工作区内的相对路径（如 data/report.xlsx）， value 为 FileEntry 对象： 

{
  "data/report.xlsx": {
    "file_uri": "tos://bucket/uploads/user_123/a1b2c3d4.xlsx",  // TOS 存储地址
    "sha256": "e3b0c44298fc1c149afb...",                        // 文件内容哈希，用于去重与校验
    "is_dir": false                                              // true 表示目录节点
  },
  "charts/": {
    "file_uri": "",
    "sha256": "",
    "is_dir": true
  }
}

skills 字段结构（skill_id → SkillEntry）

skills 是一个 Map，key 为技能唯一标识，value 为该工作区中技能的配置状态： 

{
  "skill_stock_analysis": {
    "enabled": true,              // 是否在此工作区中启用
    "loaded_version": "1.2.0"    // 当前工作区加载的技能版本号
  },
  "skill_pdf_parser": {
    "enabled": false,
    "loaded_version": "0.9.1"
  }
}

agent_sessions

会话集合记录用户与 Agent 的每一次对话会话的元数据。 同一会话同时只能处理一个任务——当会话处于 running 状态时，新的请求将被拒绝，直至当前任务完成或被取消。

字段说明

字段名 类型 说明
_id ObjectId 会话唯一标识
user_id String 会话所属用户 ID
workspace_id ObjectId 关联的工作区 ID
title String 会话标题，首条消息发送后由 LLM 自动生成摘要标题
status String 会话状态枚举，见下方说明
message_count Int 消息总数，冗余存储，用于快速分页展示
last_message_at DateTime 最后一条消息的时间，用于会话列表排序
created_at DateTime 会话创建时间（UTC）
updated_at DateTime 会话最后更新时间（UTC）

status 枚举

枚举值 含义 说明
active 空闲中 会话处于等待状态，可接受新的用户消息
running 执行中 Agent 正在处理任务，拒绝新请求，直至完成或取消
deleted 已删除 软删除状态，数据保留但不对用户展示，可用于审计

索引

索引字段 类型 用途
user_id + created_at 复合索引（降序） 按用户分页查询会话列表，按创建时间倒序排列
user_id + status 复合索引 快速判断用户是否有正在运行的会话

agent_messages

消息集合是系统中最重要的集合，存储所有会话中每一条消息的完整内容。 消息内容（content）采用多态 Part 数组设计，可同时表达文本、文件、 思考过程、工具调用及工具结果等多种内容类型，完整还原 Agent 的推理与执行轨迹。

基础字段说明

字段名 类型 说明
_id ObjectId 消息唯一标识
session_id ObjectId 所属会话 ID，关联 agent_sessions._id
user_id String 消息所属用户 ID，冗余存储，便于跨会话查询
role String 消息角色：user / assistant / system
content Array<Part> 消息内容数组，由多个 Part 组成，支持多种类型混合（见下方 Part 类型说明）
metadata Object 扩展元数据，如 token 用量、模型名称、压缩摘要标记等
created_at DateTime 消息创建时间（UTC）
updated_at DateTime 消息最后更新时间（UTC），流式输出完成后更新

content 数组 Part 类型

每条消息的 content 字段是一个 Part 数组，通过 type 字段区分类型。 以下是所有支持的 Part 类型及其 JSON 格式：

TextPart — 文本内容
{
  "type": "text",
  "text": "根据您上传的财报数据，Q3 营收同比增长 23.5%，主要驱动力来自..."
}
FilePart — 文件附件
{
  "type": "file",
  "file_uri": "tos://finance-agent-bucket/uploads/user_123/2024q3_report.xlsx",
  "name": "2024q3_report.xlsx",
  "mime_type": "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
  "size": 204800
}
ThinkingPart — 模型思考过程（扩展思考）
{
  "type": "thinking",
  "thinking": "用户想要分析 Q3 财报，我需要先读取 Excel 文件中的收入数据，\n然后计算同比增长率，最后生成可视化图表..."
}
ToolUsePart — 工具调用请求
{
  "type": "tool_use",
  "id": "call_abc123xyz",
  "name": "execute_python",
  "input": {
    "code": "import pandas as pd\ndf = pd.read_excel('/workspace/2024q3_report.xlsx')\nprint(df.head())"
  }
}
ToolResultPart — 工具执行结果
{
  "type": "tool_result",
  "tool_use_id": "call_abc123xyz",
  "tool_name": "execute_python",
  "status": "success",
  "content": [
    {
      "type": "text",
      "text": "   Quarter  Revenue  YoY_Growth\n0       Q1   1250.3       18.2%\n1       Q2   1380.7       21.1%\n2       Q3   1542.9       23.5%"
    }
  ],
  "duration_ms": 1340
}

status 枚举值：success（执行成功）/ error（执行失败）/ timeout（执行超时）。
当 status 为 error 时，content 中包含详细的错误信息， Agent 会根据错误信息自动重试或调整策略。

索引

索引字段 类型 用途
session_id + created_at 复合索引（升序） 按会话拉取消息历史，按时间顺序排列
user_id + created_at 复合索引（降序） 跨会话查询用户消息历史

agent_skills

技能集合管理所有已注册的 Agent 技能包，包括内置技能和用户自定义技能。 每个技能可有多个版本（versions 数组），支持版本管理与灰度发布。

字段说明

字段名 类型 说明
_id String 技能唯一标识，人类可读，如 skill_stock_analysis
name String 技能展示名称，如"股票分析助手"
description String 技能功能描述，注入 LLM System Prompt 帮助模型决策是否使用该技能
cur_version String 当前稳定版本号（SemVer），新工作区默认加载此版本
type String 技能类型：builtin（内置）/ custom（用户自定义）
owner_id String 技能创建者用户 ID，内置技能为系统保留值 system
scope String 可见范围：public（所有用户）/ private（仅创建者）/ team（团队内）
scope_id String 当 scope 为 team 时，记录对应的团队 ID
versions Array<VersionEntry> 历史版本列表，每个版本记录版本号、TOS 包路径、发布时间及变更说明
status String 技能状态：active（可用）/ deprecated（已废弃）/ draft（草稿）

agent_files

文件集合存储用户上传文件和 Agent 产物的元数据，实际二进制内容存储于 TOS。 元数据与 TOS 路径解耦，支持文件重命名、移动而不修改 TOS 存储位置。

字段名 类型 说明
_id ObjectId 文件唯一标识
user_id String 文件所属用户 ID
workspace_id ObjectId 文件所在工作区 ID
session_id ObjectId 文件来源会话 ID（可选，Agent 产物关联会话）
name String 文件原始名称
file_uri String TOS 存储路径，格式见路径规范
mime_type String 文件 MIME 类型，如 application/vnd.ms-excel
size Int64 文件大小（字节）
sha256 String 文件内容 SHA-256 哈希，用于完整性校验与去重
source String 文件来源：upload（用户上传）/ artifact（Agent 产物）
created_at DateTime 文件创建时间（UTC）

agent_shares

分享集合管理会话的外部分享记录，用户可将特定会话生成分享链接， 通过 token 实现无需登录的只读访问。

字段名 类型 说明
_id ObjectId 分享记录唯一标识
session_id ObjectId 被分享的会话 ID
owner_id String 分享创建者用户 ID
token String 分享访问令牌（URL-safe 随机字符串，32 字节），用于构造分享链接
expires_at DateTime 分享链接过期时间，null 表示永不过期
view_count Int 链接访问次数统计
status String 分享状态：active（有效）/ revoked（已撤销）
created_at DateTime 分享创建时间（UTC）

agent_traces

追踪集合记录每次 Agent 任务执行的详细追踪信息，用于性能分析、问题排查与用量统计。 每条追踪记录对应一次完整的用户请求到 Agent 响应的全过程。

字段名 类型 说明
_id ObjectId 追踪记录唯一标识
session_id ObjectId 关联会话 ID
user_id String 执行用户 ID
trigger_message_id ObjectId 触发本次执行的用户消息 ID
loops Int Agent 主循环迭代次数
tool_calls Array<Object> 本次执行中所有工具调用的摘要（工具名、耗时、状态）
total_input_tokens Int 本次执行累计输入 Token 数（所有轮次之和）
total_output_tokens Int 本次执行累计输出 Token 数（所有轮次之和）
duration_ms Int64 任务总耗时（毫秒），从接收用户消息到发出 done 事件
status String 执行结果：completed（正常完成）/ cancelled（用户取消）/ error（异常终止）
error String 当 status 为 error 时记录错误详情
created_at DateTime 追踪记录创建时间（UTC）

TOS 存储路径规范

所有存入 TOS 的文件均遵循统一的路径命名规范，确保按业务类型隔离存储、 便于权限管控与生命周期策略配置（如过期清理）。

TOS Bucket 根地址：tos://finance-agent-bucket/

类型 路径规范 说明
用户上传文件 uploads/{user_id}/{file_id}/{filename} 用户通过前端或 API 上传的原始文件（Excel、CSV、PDF 等），按用户 ID 隔离
Agent 产物 artifacts/{user_id}/{session_id}/{filename} Agent 执行代码后生成的输出文件（图表、报告等），按会话维度组织，便于关联溯源
Skills 技能包 skills/{name}/{version}.zip 技能开发者发布的技能压缩包，按技能名称和版本号组织，支持多版本共存

路径示例

# 用户上传的财报文件
tos://finance-agent-bucket/uploads/user_ByteUser_88881234/a1b2c3d4e5f6/2024q3_report.xlsx

# Agent 生成的营收趋势图
tos://finance-agent-bucket/artifacts/user_ByteUser_88881234/665f1a2b3c4d5e6f7a8b9c01/revenue_trend.png

# 股票分析技能包 v1.2.0
tos://finance-agent-bucket/skills/skill_stock_analysis/1.2.0.zip

安全策略：所有文件访问均通过 Pre-signed URL 鉴权， 签名有效期默认 1 小时，到期后链接自动失效，防止未授权访问。 前端展示文件时，后端实时生成签名 URL 返回，不在数据库中持久化签名链接。
